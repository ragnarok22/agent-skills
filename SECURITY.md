# Security Policy

## Supported Versions

Security fixes are applied to the latest commit on the repository's default
branch.

| Version                         | Supported |
| ------------------------------- | --------- |
| Latest commit on default branch | Yes       |
| Older commits or tags           | No        |

## Reporting a Vulnerability

Please report suspected vulnerabilities privately. Do not open a public issue
for security reports.

Send reports to: `sasuke.reinier@gmail.com`

Include the following details when possible:

- A clear description of the issue and potential impact
- Affected files, paths, and versions/commits
- Reproduction steps or a proof of concept
- Any suggested mitigations

## Response Process

- Initial acknowledgment target: within 3 business days
- Status update target: at least every 7 business days while triaging
- Remediation timeline: depends on severity and complexity

After a fix is ready, maintainers may coordinate disclosure timing with the
reporter.
